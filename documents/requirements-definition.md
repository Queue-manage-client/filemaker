## 1. システム概要

### 1.1 システム名称

**Dispatch Harmony Hub**（ディスパッチ・ハーモニー・ハブ）

### 1.2 システム概要

本システムは、派遣型サービス業における業務管理を一元化するWebアプリケーションである。店舗運営、スタッフ管理、ホステス（キャスト）管理、顧客管理、予約・配車管理、会計・給与管理など、事業運営に必要な全ての業務プロセスをサポートする。

### 1.3 システム構成

| 項目 | 内容 |
|------|------|
| システム形態 | Webアプリケーション（SPA） |
| フロントエンド | Next.js 15 / React 18 / TypeScript |
| バックエンド | Supabase（PostgreSQL） |
| 認証基盤 | Supabase Auth |
| ホスティング | Vercel（予定） |

---

## 2. 目的と背景

### 2.1 背景

派遣型サービス業において、以下の課題が存在していた：

- 複数のExcelファイルや紙ベースでの管理による非効率
- リアルタイムでの状況把握が困難
- 店舗間・スタッフ間での情報共有の遅延
- 会計処理・給与計算の煩雑さ
- 顧客情報の散逸による機会損失

### 2.2 目的

1. **業務効率化**: 全業務プロセスのデジタル化による作業時間短縮
2. **リアルタイム管理**: 配車状況・出勤状況のリアルタイム可視化
3. **データ一元管理**: 顧客・スタッフ・売上データの統合管理
4. **意思決定支援**: 各種集計・分析機能による経営判断のサポート
5. **セキュリティ強化**: 役割ベースのアクセス制御による情報保護

### 2.3 期待効果

| 項目 | 期待効果 |
|------|----------|
| 配車業務 | 配車決定時間 50%短縮 |
| 日報作成 | 日報作成時間 80%短縮 |
| 顧客対応 | 顧客情報検索時間 90%短縮 |
| 経理業務 | 月次締め処理時間 60%短縮 |

## 4. 機能要件

### 4.1 リアルタイム運用管理

#### 4.1.1 RTパネル（リアルタイムパネル）

| 項目 | 内容 |
|------|------|
| 機能概要 | 現在の配車状況・予約状況をリアルタイムで表示 |
| 主要機能 | 予約一覧表示、配車状況確認、ドライバー位置表示 |
| 更新頻度 | 自動更新（30秒間隔） |
| 対象ユーザー | 配車担当スタッフ、管理者 |

**詳細要件**:
- 時間軸に沿った予約の視覚的表示
- IN/OUT待ち予約の区別表示
- ホステス別・ドライバー別のフィルタリング
- 緊急予約のハイライト表示

| 項目 | 内容 |
|------|------|
| 機能概要 | 2次元マップ形式での配車管理 |

**詳細要件**:
- 地図上でのホテル・顧客位置表示

- 交通費自動計算
